Un excellent guide pour obtenir un hardmod se trouve [ici] (https://gbatemp.net/threads/414498/). Pour les New 3DS, lire [ceci] (https://github.com/Plailect/Guide/issues/681) avant de commencer le hardmod.
{: .notice--info}

Il s'agit d'une implémentation fonctionnelle de l'exploit "FIRM partitions known-plaintext" décrit [ici](https://www.3dbrew.org/wiki/3DS_System_Flaws).
{: .notice--info}

Cela fonctionne sur les New 3DS, Old 3DS et 2DS.
{: .notice--success}

#### Instructions

##### Section I - Modification de la NAND

1. Extrayez le `.zip` d'autofirm dans un dossier nommé `autofirm`
2. Placez une copie de la sauvegarde de votre NAND (appelée `nand.bin`) dans le dossier `/autofirm/`
3. Copiez les éléments du `.zip` de NFIRM dans le dossier `/autofirm/source/firmwares/`
4. Lancez `autofirm.bat` et sélectionnez la console et la version qui convient au backup NAND
5. Patientez pendant l’exécution du script
6. Si tout a fonctionné, alors votre backup NAND original aura été renommé `nand.bin` et vous aurez un `nand_patched.bin` modifié contenant le NATIVE_FIRM pour 10.4.0
  + Gardez `nand.bin`, il sera utilisé plus tard
7. Flashez ce `nand_patched.bin` sur votre console grâce à votre hardmod

##### Section II - Vérification de l'exploit

1. Copiez et fusionnez le dossier `3ds` de l'archive `.zip` de 3DSident à la racine de votre carte SD
2. Réinsérez votre carte SD dans la console
3. Utilisez [Homebrew Launcher (SoundHax)](homebrew-launcher-(soundhax)) pour lancer le homebrew launcher sur la console
4. Lancez 3DSident
5. Vérifiez les éléments suivants:
  + **Kernel version**: 2.50-11
  + **FIRM version**: 2.50-11
  + Si l'un ou l'autre n'affiche pas les versions ci-dessus, quelque chose s'est mal passé et vous devez réessayer depuis le début

Le numéro de version n'aura *pas* changé dans vos paramètres, mais l'exploit a bien fonctionné.
